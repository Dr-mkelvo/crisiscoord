# CrisisCoord UI/UX Principles

## Design Goal

Make a judge immediately understand that CrisisCoord is not a chatbot. It is a shared crisis operations room where specialized agents coordinate, block, review, and escalate decisions.

## Priority References

- Figma Dashboard UI templates for layout inspiration.
- shadcn/ui for code-aligned component patterns.
- Radix Primitives for accessible interaction behavior.
- Component Gallery and UI Guideline for component naming and pattern checks.
- Material Design and Apple HIG for clarity, states, and adaptive behavior.
- Laws of UX for cognitive load and decision simplification.

## Screen Priorities

1. Crisis Command Room.
2. Agent Handoff Map.
3. Evidence And Audit View.
4. Communications Review.
5. Mobile Decision Queue.

## Critical UX Requirements

- Show the current incident severity and deadline at all times.
- Show each agent's state.
- Show dependency gates.
- Show last updated timestamps.
- Show generated drafts as drafts.
- Show human decisions as explicit review actions.
- Show source facts for generated communications.
- Show failures and retry options.

## Avoid

- Marketing hero pages.
- Generic AI chat layouts.
- Unclear license assets.
- Decorative 3D assets.
- Copying third-party templates directly.
- Hidden automation with no room-state explanation.
