# Presentation Packet

Last updated: June 13, 2026.

This folder holds the Gamma-ready presentation materials for the CrisisCoord hackathon submission.

## Files

- [gamma-presentation-plan.md](./gamma-presentation-plan.md): deck strategy, Gamma workflow, architecture caveat, and team meeting questions.
- [gamma-deck-brief.md](./gamma-deck-brief.md): paste-ready Gamma prompt with the full story, design direction, and card-by-card content.
- [visual-storyboard.md](./visual-storyboard.md): visual system, slide rhythm, and the proof objects Gamma should render.

## Rule

The deck may explain the architecture concept, using the master implementation guide as the source of truth. It must clearly distinguish what is built, seeded, planned, or not part of the demo.

## Gamma Workflow

1. Open Gamma.
2. Start from pasted text.
3. Paste the prompt in [gamma-deck-brief.md](./gamma-deck-brief.md).
4. Use [visual-storyboard.md](./visual-storyboard.md) as the manual polish checklist.
5. Review all compliance and architecture claims manually.
6. Add app screenshots only after the UI exists.
7. Export final slides to PDF/PPT and keep the Gamma link for submission.

## Quality Bar

The final deck should feel like a calm enterprise crisis command room, not a spreadsheet screenshot. Avoid giant KPI tiles, fake dashboard numbers, generic AI robot imagery, clipart, and heavy tables. The strongest slides should show handoffs, dependency gates, agent room state, human approval, and auditability.
